# Dashcam Video Joiner

コムテック製ドライブレコーダーの AVI 動画ファイルを 1 本に結合する Windows 用ツールです。

## 目的

前方カメラで録画された 30 秒単位の AVI ファイルを、音ズレリスクを抑えながら結合します。
**デフォルトは「音ズレ対策モード」で、出力は MP4 です。**

> **注意**: 音ズレ対策モードでも完全に音ズレしないことを保証するものではありません。

---

## 対応環境

- OS: Windows 11
- Python: 3.9 以上 (Python 実行時)
- GUI: Tkinter (Python 標準ライブラリ)
- 外部ツール: ffmpeg / ffprobe (別途配置が必要)

---

## 対応ファイル名

```
YYYYMMDD_HHMMSS_F_Nor.AVI

例:
  20260605_071930_F_Nor.AVI
  20260605_072000_F_Nor.AVI
  20260605_072031_F_Nor.AVI   ← 31秒間隔も許容 (29〜32秒)
```

前方カメラ (`Front` フォルダ) の動画を対象としてください。

---

## 録画空白区間の扱い

駐車・休憩・エンジン停止などでドラレコ録画が一時停止すると、ファイル名の時刻に数分以上の空白が生じることがあります。
GUI の「録画空白区間の扱い」で以下の3択から選べます。

| 設定 | 挙動 |
|------|------|
| **警告して1本に結合する（推奨）** | 空白を検出しログに警告を出して処理を続行。空白時間ぶんの黒画面・無音は挿入しない。クリップを時系列にそのまま繋ぐ |
| **空白区間で分割して複数ファイルにする** | 空白の前後でセグメント分割し、複数の出力ファイルを生成する。ファイル名は `_part001_YYYYMMDD_HHMMSS-YYYYMMDD_HHMMSS` 形式 |
| **空白区間があれば中止する（厳密チェック）** | 空白区間を検出した時点でエラー停止。完全に連続した録画だけを結合したい場合に使う |

> **注意 — 時刻差が 29秒未満の場合（詳細判定あり）**
>
> エンジン停止時などに、最後のファイルが 30 秒未満（例: 10 秒）で終わることがあります。
> その後すぐ録画が再開すると、ファイル名の時刻差が 29 秒未満になる場合があります。
> このツールでは、前ファイルの実 duration（映像ストリーム長）を確認して以下のように扱います。
>
> | 条件 | 扱い |
> |---|---|
> | `前ファイルduration ≤ 時刻差 + 1.0秒` | 「短い最終クリップ後の録画再開」として録画空白と同じ扱い |
> | `前ファイルduration > 時刻差 + 1.0秒` | 重複・順序異常の疑いとしてエラー停止 |
> | 前ファイルduration 取得不可 | 安全側でエラー停止 |
>
> 時刻差が **0秒以下**（重複・時計異常の疑い）は常にエラー停止します。
> 空白時間ぶんの黒画面や無音は挿入しません。

## 結合モード

### ● 音ズレ対策モード（推奨・デフォルト）

| 項目 | 内容 |
|------|------|
| 出力形式 | **MP4** |
| 処理方式 | 各 AVI を個別に MP4 へ正規化 → 正規化済み MP4 を結合 |
| 映像 | H.264 (libx264, CRF 18, preset veryfast) |
| 音声 | AAC 192k / 48000Hz |
| タイムスタンプ | `setpts=PTS-STARTPTS` + `aresample=async=1:first_pts=0` でリセット |
| 中間ファイル | `work/normalize_*/` に一時保存。成功時は自動削除（オプションで保持可） |
| 処理時間 | 再エンコードのため遅い (2時間分で数分〜十数分程度) |

実ファイルで `-c copy` 方式にて音ズレが発生したため、このモードをデフォルトとしています。

### ○ 高速・無劣化モード（非推奨）

| 項目 | 内容 |
|------|------|
| 出力形式 | **AVI** |
| 処理方式 | AVI を `-c copy` で直接結合 (再エンコードなし) |
| 速度 | 高速 |
| **注意** | **コムテック AVI では音ズレが発生する可能性があります** |

## ffmpeg / ffprobe の配置方法

1. [ffmpeg 公式サイト](https://ffmpeg.org/download.html) または
   [gyan.dev ビルド](https://www.gyan.dev/ffmpeg/builds/) から
   Windows 向けビルドを入手します。
2. `tools/README.md` の手順に従い、`ffmpeg.exe` と `ffprobe.exe` を
   `tools/` フォルダに配置します。

```
dashcam-video-joiner/
  tools/
    ffmpeg.exe     ← ここに置く
    ffprobe.exe    ← ここに置く
```

システムの PATH に ffmpeg が登録されている場合は配置不要です。
ツールは `tools/` → 実行ファイルと同階層 → PATH の順に検索します。

## 使い方

1. **結合モードを選択** — 音ズレ対策モード（推奨）または高速・無劣化モード
2. **動画ファイルを選択** — 結合したい AVI ファイルを複数選択
3. **出力先を選択** — 出力ファイルの保存先を指定（モードに応じた拡張子）
4. **中間ファイルを残す** — チェックするとデバッグ用に中間 MP4 を保持（音ズレ対策モードのみ）
5. **結合開始** — チェックと結合が自動で実行される
6. 完了後、`logs/` フォルダに詳細ログが保存される

### ログとフォルダの場所

| 項目 | 場所 |
|------|------|
| ログ (Python実行時) | `<プロジェクトルート>/logs/join_YYYYMMDD_HHMMSS.txt` |
| ログ (exe実行時) | `dist\DashcamVideoJoiner\logs\join_YYYYMMDD_HHMMSS.txt` |
| 中間ファイル | `<プロジェクトルート>/work/normalize_YYYYMMDD_HHMMSS/` |

## 音ズレ対策として行っていること

**音ズレを完全に保証するものではありません。**
以下の対策で音ズレリスクを低減しています。

| 対策 | 内容 |
|------|------|
| ファイル名形式チェック | 不正なファイルを事前に検出 |
| 時刻差チェック (29〜32秒) | ファイル欠けや順序ミスを検出 (30秒・31秒混在に対応) |
| ffprobe ストリーム検査 | 動画・音声トラックの存在と仕様を確認 |
| 全ファイルの仕様一致チェック | codec / 解像度 / fps / sample rate / channels の不一致を検出 |
| stream duration 差チェック | 各ファイルの video/audio stream duration 差が 0.5 秒超でエラー停止 |
| 出力先上書き防止 | 出力先が入力ファイルと同じ場合は停止 |
| **AVI→MP4 正規化 (音ズレ対策モード)** | PTS をリセットし音声タイムスタンプをならす |
| **音声 AAC 再エンコード (音ズレ対策モード)** | 最終結合時も `aresample=async=1` で再同期 |
| 結合後 duration 比較 | 入力合計と出力の長さを比較 (差 2 秒以内で OK)。**音ズレ対策モード**では正規化後 MP4 の duration 合計を基準とする（元 AVI の `format.duration` は実際の映像長より長い場合があるため）。**高速・無劣化モード**では元 AVI の `format.duration` 合計を基準とする |
| 詳細ログ保存 | 全ファイルの ffprobe 結果・ffmpeg stderr を記録 |

## 既知の制限

- **音ズレ対策モードでも完全に音ズレしないことは保証しません**
- 初期版では **AVI 入力のみ** 対応
- **Front フォルダ内の前方カメラ動画のみ** を対象とした設計
- リアカメラ動画との結合には非対応
- ドラッグ＆ドロップ非対応
- **元の AVI ファイルは編集・削除しません**
- 映像 1 本・音声 1 本のみを結合します。
  GPS データや独自データストリームは出力に保持されない場合があります
- 音ズレ対策モードは中間 MP4 を作成するため、処理時間と一時ディスク容量が増えます
  (2時間分で元 AVI とほぼ同容量の一時領域が必要)
- 高速・無劣化モードはコムテック AVI で音ズレする可能性があります

## トラブルシューティング

### ffmpeg/ffprobe が見つからないと表示される

- `tools/ffmpeg.exe` と `tools/ffprobe.exe` が存在するか確認してください。
- または ffmpeg を PATH に追加してください。

### ファイル名形式エラー

- ファイル名が `YYYYMMDD_HHMMSS_F_Nor.AVI` 形式であることを確認してください。
- `Front` フォルダ内のファイルのみ選択してください。

### 時刻差エラー (29秒未満)

- ファイルが重複していないか確認してください。
- ファイルの順序に異常がないか確認してください。
- ドライブレコーダーの時計設定を確認してください。

### 録画空白区間で止まってしまう

- 「録画空白区間の扱い」が **厳密チェック** になっている場合、空白を検出するとエラー停止します。
- **警告して1本に結合する（推奨）** または **空白区間で分割して複数ファイルにする** に切り替えてください。
- 32秒を超える間隔（駐車・休憩など）は空白区間として扱われます。

### 出力先が入力ファイルと同じというエラー

- 出力ファイル名を入力ファイルと別のファイル名・フォルダに指定してください。

### 結合後 duration 警告

- 軽微な場合は正常範囲内のことがあります。
- 大きくずれている場合は、ログファイルで詳細を確認してください。

### 音ズレ対策モードで正規化が失敗する

- `work/normalize_*/` フォルダの中間ファイルと `logs/` のログを確認してください。
- ffmpeg が libx264 をサポートしているか確認してください（フルビルドが必要）。
